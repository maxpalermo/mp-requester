Manuale componente <web-requester>

<web-requester> è un Web Component (Custom Element) che mostra un dialog modale per richiedere una conferma all’utente. Espone un’API asincrona basata su Promise<boolean> (risolve true o false).

File sorgente: views/asset/js/web-requester.js

1) Inclusione in una pagina web

Assicurati di caricare lo script del componente prima di usarlo.

<script src="/modules/mpexportdocuments/views/asset/js/web-requester.js"></script>

Il percorso può variare in base alla pagina (front/admin) e al modo in cui includi gli asset in PrestaShop.

2) Inserimento del tag

Opzione A (consigliata): singleton

Non devi aggiungere manualmente il tag in HTML: le chiamate statiche (es. WebRequester.confirm) creano automaticamente un’istanza se non esiste. 

// apre un dialog e risolve true/false
const ok = await WebRequester.confirm({
  title: "Conferma azione",
  message: "Sei sicuro?",
  type: "warning"
});
Opzione B: istanza manuale

Puoi creare e appendere tu un’istanza, poi usare i metodi d’istanza.

const wr = document.createElement("web-requester");
document.body.appendChild(wr);

const ok = await wr.confirm({
  title: "Conferma",
  message: "Procedo?",
  type: "info"
});

Evita di usare contemporaneamente singleton + istanza manuale nella stessa pagina, a meno che non sia voluto.

3) Attributi

Attributo Valori Descrizione
theme soft
bold
pastel
elegant
glass 		Seleziona la palette di stile generale. Se non presente (o non valido) viene usato il default (elegant).

4) Proprietà e metodi principali (istanza)

Nome Firma Descrizione
theme wr.theme = "soft" Getter/setter dell’attributo theme.
setTheme wr.setTheme(theme) Alias pratico per impostare il tema.
confirm wr.confirm(options) => Promise<boolean> Apre un dialog di conferma e risolve true/false.
info wr.info(options) => Promise<boolean> Shortcut che forza type: "info" e titolo di default “Informazione”.
success wr.success(options) => Promise<boolean> Shortcut che forza type: "success" e titolo di default “Operazione completata”.
warning wr.warning(options) => Promise<boolean> Shortcut che forza type: "warning" e titolo di default “Attenzione”.
danger wr.danger(options) => Promise<boolean> Shortcut che forza type: "danger" e titolo di default “Errore”.

5) Metodi statici (singleton)

I metodi statici usano un’istanza singleton. Se <web-requester> non è presente in pagina, viene creato e appeso a document.body. 

// equivalentemente ai metodi d’istanza
await WebRequester.confirm({ title: "Conferma", message: "OK?" });
await WebRequester.info({ message: "Messaggio informativo" });
await WebRequester.success({ message: "Operazione completata" });
await WebRequester.warning({ message: "Attenzione" });
await WebRequester.danger({ message: "Errore" });

6) Opzioni dei metodi (options)

Campo Tipo Default Note
title string "Conferma" Titolo mostrato nella barra superiore.
message string "" Testo del messaggio (supporta a-capo).
type "info" | "success" | "warning" | "danger" "info" Determina colore barra superiore e stile del bottone “Si”.
theme "soft" | "bold" | "pastel" | "elegant" | "glass" undefined Se passato, forza la palette per quella invocazione (imposta data-theme sul componente). Se omesso, viene usato il tema dell’istanza (attributo theme) o il default.

7) Comportamento e ritorno

Il metodo risolve sempre una Promise<boolean>:

Azione utente Risultato
Clic su “Si” true
Clic su “No” false
Esc o chiusura del dialog (es. cancel) false

È previsto un controllo per impedire richieste concorrenti: se chiami confirm() mentre un’altra è pendente, la Promise viene rigettata con errore.

8) Esempio pratico (integrazione in una funzione)

async function exportXML(idOrder) {
  const ok = await WebRequester.warning({
    title: "Conferma esportazione",
    message: "Confermi di voler esportare il documento in XML?",
    theme: "soft"
  });

  if (!ok) return;

  // ... esegui fetch / azione di export
}

9) Troubleshooting

Problema Causa tipica Soluzione
I dialog “spariscono” in sequenza Gestione eventi close / race quando si aprono più dialog di seguito Usa la versione aggiornata del componente (include fix su close).
Non appare nulla Script non caricato / path errato Verifica che window.WebRequester sia definito in console.
